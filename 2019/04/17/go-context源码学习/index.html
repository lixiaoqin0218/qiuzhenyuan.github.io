Golang中的context

context是什么

context 是go1.7引入标准包的接口
context接口需要实现4个方法：

分别解析一下这context接口的4个方法：

Deadline()
返回 time.Time 和 ok。如果没有设置deadline, ok=false,否则ok=true。 time.Time返回的是过期时间。

Done()
当代表此上下文完成的工作应该被取消时，Done会返回一个已关闭的频道;如果永远不能取消此上下文，则Done()可能返回nil。
WithCancel()安排在Cancel()方法被调用时返回关闭的channel;
WithDeadline()安排在deadline到时返回关闭到channel;
WithTimeout()安排在超时时返回关闭的channel。

一般Done()方法是这样使用的：

Done()返回只读channel。在goroutine中，使用select调用<-ctx.Done()判断是否要结束，如果接受到值的话，就可以返回结束goroutine了；如果接收不到，就会继续进行监控。

Err()
当context的Done()方法未被调用时，Err()返回nil,当Done()被调用，返回具体原因。

Value()
Value()方法返回context中与key相关联的value。

context有什么用

摘抄一段 src/context/context.go的注释

// Incoming requests to a server should create a Context, and outgoing
// calls to servers should accept a Context. The chain of function
// calls between them must propagate the Context, optionally replacing
// it with a derived Context created using WithCancel, WithDeadline,
// WithTimeout, or WithValue. When a Context is canceled, all
// Contexts derived from it are also canceled.

一个请求进入服务端，应该创建一个context。对外提供服务的服务端，应该接收一个context。它们之间的调用应该传播context,有需要可以用 WithCancel(),WithDeadline(),WithTimeout()方法创建派生的context。当一个context被取消，由它派生的context都会被取消

使用context的规范

// Programs that use Contexts should follow these rules to keep interfaces
// consistent across packages and enable static analysis tools to check context
// propagation:
//
// Do not store Contexts inside a struct type; instead, pass a Context
// explicitly to each function that needs it. The Context should be the first
// parameter, typically named ctx:
//
// func DoSomething(ctx context.Context, arg Arg) error {
// // … use ctx …
// }
//
// Do not pass a nil Context, even if a function permits it. Pass context.TODO
// if you are unsure about which Context to use.
//
// Use context Values only for request-scoped data that transits processes and
// APIs, not for passing optional parameters to functions.

  1. 不要把context存在结构体中, 而应该把它作为函数的首个参数。
  2. 不要传递 nil Context。
  3. context 应该只用来传递请求域的参数，不应该用来保存函数的可选参数。

源码中实现的几种context

context有三种, emptyCtx, cancelCtx,timerCtx和 valueContext

emptyCtx

emptyCtx实际上是int类型，实现了 context接口的4个方法，但是没有任何逻辑。emptyCtx不会被取消，所以Err()方法返回nil

Background()方法和TODO()方法返回的就是emptyCtx类型的context,两个函数返回的context本质上是一样的，都是不可取消，没有过期时间，一般作为context树的顶层。他们的区别是Background()一般位于main函数中，初始化的时候或者在 tests中，在request进入的时候生成；而TODO()一般是不清楚使用什么context时，使用的 。
